- 服务类普遍采用单例模式（如 `AuthService`, `WebSocketService`, `DataLayer`），通过私有构造函数和静态 `instance` 字段实现。
- API 响应统一封装为 `ApiResponse` 对象，包含 `success`, `data`, `message` 字段，由 `ApiClient` 统一处理。
- 本地缓存键名采用领域前缀格式（如 `user:123:profile`, `conv:42:list`），便于 `DataLayer` 进行批量失效和 TTL 管理。
- Riverpod Provider 定义在对应的 Notifier 文件底部，并通过 `sharedPreferencesProvider` 进行依赖注入。
